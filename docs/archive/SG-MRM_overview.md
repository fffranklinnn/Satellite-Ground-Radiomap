# 卫星下行链路多尺度电磁地图 (SG-MRM) 开发指南

## 1. 项目愿景
本项目旨在构建一个基于“瓦片叠加（Tile Stacking）”逻辑的仿真系统。所有物理层级（L1/L2/L3）均输出标准化的 **256×256 像素图片**，每一像素代表对应尺度下的电磁损耗（dB）。通过这种方式，实现从星端发射到地面接收的全链路、多分辨率动态映射。

## 2. 核心架构：三级瓦片设计
系统逻辑遵循“物理现象尺度决定层级”的原则：

| 层级            | 物理覆盖范围 | 像素分辨率 | 物理含义 (m/px) | 核心考量因素                                          |
| :-------------- | :----------- | :--------- | :-------------- | :---------------------------------------------------- |
| **L1 (广域层)** | 256 km       | 256×256    | 1000 m          | **卫星TLE、天线能量图、电离层(P.531)、大气层(P.618)** |
| **L2 (地形层)** | 25.6 km      | 256×256    | 100 m           | **详细山地数据 (DEM)**、地物统计损耗                  |
| **L3 (微观层)** | 256 m        | 256×256    | 1 m             | **建筑物分布**、材质属性、射线追踪(反射/衍射)         |

## 3. 阶段性预期目标 (敏捷迭代)

### A. 中间版本 (V1.0)：静态链路闭环
**预期成效：**
- **L1**: 实现基于 TLE 的卫星定位和简单的天线增益投影。
- **L3**: 实现基于建筑 Shapefile 的基础遮挡阴影。
- **Aggregator**: 能够将 L1 的广域能量底图与 L3 的建筑损耗矩阵进行叠加，生成第一张覆盖特定街区的 $256 \times 256$ 复合地图。

### B. 最终版本 (V2.0)：全维度动态仿真
**预期成效：**
- **L1 全物理化**：集成 1h 分辨率的时间流，同时计算**对流层降雨衰减**与**电离层闪烁**，反映实时气象。
- **L2 地形化**：引入高精度山地 DEM，解决山地城市由于地理起伏导致的信号盲区。
- **L3 高保真**：基于 GPU 射线追踪，支持多径反射。
- **最终输出**：产出随时间变化的动态 256x256 电磁地图数据集。

## 4. 模块化分工规范
- **高度内聚**：L1 关注“天空”和“能量源”，L2 关注“山地”，L3 关注“建筑”。
- **标准化接口**：每个模块必须实现 `.compute()` 接口，输入坐标原点，输出 `ndarray(256, 256)`。
- **数据解耦**：层间计算采用 dB 域加法，互不干涉物理细节。



项目结构

```
SG_RadioMap_Project/
├── configs/                # 全局配置：定义仿真原点 (Lat, Lon)、频率、时间范围等
│   └── mission_config.yaml
├── data/                   # 原始数据源 (按层分工)
│   ├── l1_space/           # TLE文件, 天线Pattern (.csv/.mat), 气象数据 (.nc)
│   ├── l2_topo/            # 山地 DEM 数据 (.tif/.hgt)
│   └── l3_urban/           # 建筑 Shapefile (.shp) 或 3D 城市模型
├── src/
│   ├── core/               # 跨层工具
│   │   ├── grid.py         # 核心：定义 256x256 的网格坐标系
│   │   └── physics.py      # 基础公式：FSPL、极化损失等
│   ├── layers/             # 核心物理层
│   │   ├── base.py         # 抽象基类 BaseLayer
│   │   ├── l1_macro.py     # 逻辑：TLE位置 -> 天线映射 -> 电离层/大气层损耗
│   │   ├── l2_topo.py      # 逻辑：DEM数据 -> 山体遮挡计算
│   │   └── l3_urban.py     # 逻辑：建筑Shapefile -> 射线追踪/阴影
│   ├── engine/
│   │   └── aggregator.py   # 叠加器：Result = L1 + Interpolate(L2) + L3
│   └── utils/              # 绘图 (PNG导出)、日志、性能分析
├── main.py                 # 程序唯一入口：通过循环处理 1h 分辨率的时间序列
└── requirements.txt
```
