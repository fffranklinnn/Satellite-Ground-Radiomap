# src 目录说明

`src/` 是 SG-MRM 的运行时源码根目录，也是项目中应优先维护和评审的主代码区域。

## 1. 目录角色

- 承担核心仿真计算（网格、物理模型、三层传播、聚合引擎）
- 提供数据加载、绘图、日志与性能分析等通用能力
- 作为 `main.py` 与 `scripts/*` 的统一调用后端

## 2. 子目录内容

```text
src/
├── core/      # 网格坐标与传播物理公式
├── layers/    # L1/L2/L3 三层物理建模实现
├── engine/    # 多层聚合引擎
├── utils/     # 加载器、校验、可视化、日志、性能工具
└── __init__.py
```

## 3. 运行链路中的位置

1. `main.py` / `scripts/*` 读取配置与输入数据
2. 调用 `src/layers` 计算各层损耗图
3. 通过 `src/engine/aggregator.py` 对齐尺度并聚合
4. 使用 `src/utils` 做输出、绘图、日志与校验

## 4. 维护建议

- 新功能优先放在 `src/`，避免在历史快照目录中分叉实现。
- 对外接口变化时，同步更新 `src/*/README.md` 与测试。
- 涉及坐标、单位、时间语义的改动，必须在注释和文档中明确说明。
